# NotesOut
## 一、项目简介
NotesOut 是一款基于大模型 LLM 的学习笔记生成工具，支持多类文件转 Markdown，再调用大模型自动生成结构化学习笔记，同时提供 GUI 可视化界面与 CLI 命令行双使用模式。
## 二、支持文件格式
可解析 .ipynb、.py、.docx、.xlsx、.md、.txt、.html；不支持老式.xls，每种文件配套专属转换依赖库。
## 三、整体模块化架构
四层分层解耦，遵循单一职责、易扩展、线程安全、代码复用等设计原则：
1. GUI 层（[gui.py](gui.py)）：可视化窗口，支持文件拖拽、API 配置、日志展示，后台线程执行任务防止界面卡死；
2. 统一转换入口（convert\[_all.py](_all.py)）：对外共用转换函数 convert_files_to_md，GUI/CLI 复用；
3. 转换器包 converters/：每种文件独立转换器，共用 base 日志回调；
4. LLM 处理层（generate\[_notes.py](_notes.py)）：构建提示词、统一封装多厂商 API 调用、保存生成笔记。
### 日志统一机制
通过可选log_fn回调区分两种输出：CLI 直接 print 打印，GUI 传入自定义日志函数写入界面日志面板。
## 四、完整业务流程
1. 用户导入多格式文件 → 校验去重；
2. 统一转换为标准 Markdown 文本；
3. 拼接文件内容构建 LLM 提示词；
4. 根据配置调用硅基流动 / DeepSeek / 智谱 AI 三家兼容 OpenAI 格式的大模型 API；
5. 获取模型输出后保存为本地 Markdown 笔记，前端弹窗提示结果。
## 五、核心接口
1. 文件转换总入口：convert_files_to_md，输出文件名 + 内容字典列表；
2. 各类单文件独立转换函数（ipynb/py/docx/xlsx/txt/html）；
3. LLM 相关：build_prompt构造提示词、call_llm_api统一请求各厂商接口；
4. 工具函数：加载本地 MD 文件、持久化保存笔记；
5. GUI 主类NotesOutApp，实现拖拽、文件选择、后台生成、日志打印等交互方法。
## 六、项目目录结构
- 根目录：GUI 启动、转换总入口、LLM 处理脚本、依赖清单、Windows 一键启动批处理；
- converters 包：分文件类型转换器 + 基础日志工具；
- _\pycache\_为自动生成缓存目录。
## 七、运行与环境
1. 搭建虚拟环境，一键安装 requirements.txt 内依赖（tkinter 内置，拖拽、文档解析、网络请求等第三方库）；
2. 两种使用方式：
  - GUI：双击 bat 批处理 / python gui.py（推荐，可视化拖拽操作）；
  - CLI：单独执行转换脚本、笔记生成脚本。
## 八、扩展能力
1. 新增文件类型：新建转换器文件、导出包、在总转换函数增加后缀分支；
2. 新增大模型厂商：在 API 配置字典补充 BaseURL、模型名、请求体，同步更新 UI 下拉选项；
3. 限制条件：新增 API 需兼容 OpenAI /v1/chat/completions 标准接口格式。
## 九、核心优势
1. 双端复用：转换、日志、LLM 逻辑 GUI/CLI 共用，减少冗余代码；
2. 高可扩展：新增文件、新增大模型无需改动主体流程；
3. 用户友好：可视化拖拽、进度日志、弹窗提示、后台异步任务不阻塞界面；
4. 标准化：统一文件转换输出、统一 API 请求 / 响应结构、统一日志输出规范。
